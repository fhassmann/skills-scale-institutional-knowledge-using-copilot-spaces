# OctoAcme Project Management — Process Docs

Welcome! This README is the entry point for OctoAcme's project management documentation. It summarizes the project lifecycle and links to each detailed process guide.

## What These Docs Are For

These documents describe how OctoAcme plans, executes, and continuously improves software projects. They are intended for new and existing team members who need to understand or contribute to OctoAcme's project management practices.

## Project Lifecycle Summary

OctoAcme follows a standardized, customer-centric project lifecycle:

1. **Initiation** — Validate the problem, set goals, identify stakeholders, and create a project one-pager.
2. **Planning** — Break work into increments, plan milestones, estimate scope, and assess risks.
3. **Execution & Tracking** — Use iterative sprints, track progress via project boards, run standups and demos, and manage risks/issues as they arise.
4. **Release & Deployment** — Follow a defined release checklist, test and verify before deployment, and document changes.
5. **Retrospective & Continuous Improvement** — Capture lessons learned, review action items, and continuously refine processes.

## Roles & Personas

| Role | Responsibility |
|------|----------------|
| **Project Manager (PM)** | Coordinates delivery, schedules, risks, and communications |
| **Product Manager (PdM)** | Defines outcomes, prioritizes backlog, and measures success |
| **Developers** | Implement features, collaborate on design and testability |
| **QA/Testing** | Validate quality and acceptance criteria |
| **Stakeholders** | Provide inputs and approvals |

See [Roles & Personas](octoacme-roles-and-personas.md) for detailed persona definitions.
